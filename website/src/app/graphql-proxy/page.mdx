# GraphQL proxy

The **`clawql-graphql`** process exposes **`http://localhost:4000/graphql`** by default. **`execute`** calls this proxy so responses can be **field-projected** before they reach the modelâ€”smaller payloads than many raw REST bodies. {{ className: 'lead' }}

## Why a second process?

**`search`** is satisfied entirely inside **`clawql-mcp`**. **`execute`** delegates to the proxy so GraphQL resolution, introspection-driven field matching, and upstream HTTP stay **isolated** from the stdio MCP lifecycle. Deployments treat both as first-class services with a stable internal URL (`GRAPHQL_URL`).

## How it maps to your API

The proxy is generated from the same **OpenAPI** (or Discoveryâ†’OpenAPI) as search, using **`@omnigraph/openapi`** in the [GraphQL Mesh](https://the-guild.dev/graphql/mesh/docs/handlers/openapi) family. Upstream calls use spec **`servers`** or **`CLAWQL_API_BASE_URL`**.

## Multi-spec and fallback

- **Merged specs:** **`execute`** uses **REST** per owning document; GraphQL is not the execution path for that mode today.
- **Single spec:** GraphQL is preferred when the schema builds; on translation or runtime errors, **`execute` falls back to REST** with the same operation map.

Pregenerated **`introspection.json`** (bundled or `CLAWQL_INTROSPECTION_PATH`) speeds field matching at MCP startup.
