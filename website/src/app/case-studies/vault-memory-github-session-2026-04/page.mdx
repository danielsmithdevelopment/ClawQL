# Case study: vault memory, GitHub, and shipping `audit`

This page walks through a **multi-turn Cursor session**: **`memory_ingest`** / **`memory_recall`** into an **Obsidian vault**, **GitHub** triage and **new issues**, **backlog ordering**, and shipping the first **enterprise** vertical slice — optional **`audit`** ([#89](https://github.com/danielsmithdevelopment/ClawQL/issues/89)) — with **docs, Helm, website, and Cursor skill** alignment.

## Why this session mattered

Product and engineering context was scattered across **threads, tools, and documents**. The vault became the **durable narrative** (roadmaps, analyses, prioritization snapshots); **GitHub** became the **execution layer** (issues, epics, PRs). Without that split, you either **re-litigate** design in every chat or **lose** the reason a decision was made.

**Three optional layers** matter for this story:

- **`cache()`** — ephemeral scratch; gone when the process restarts.
- **`audit()`** — in-process operator trail; **not** the Obsidian vault.
- **`memory_ingest` / `memory_recall`** — Markdown + graph (+ optional **`memory.db`** hybrid stack).

## Narrative timeline (April 2026)

1. **Ingest wave** — External and generated content (e.g. release cross-checks, Agent vision, gRPC/Gallery, workflow tips) with **stable note titles** and often **`append: true`** so related updates **merge** instead of splintering.
2. **Tracker sync** — Issue list hygiene: e.g. **`iac_inspect`** duplicate **#39** → canonical **#69**; new tracking **#88–#91** for gateway, enterprise epic, synthetics, Gallery fork work.
3. **Prioritization** — Choose **#89** as a **bounded** delivery: one enterprise tool **end-to-end** (code + tests + ops + docs).
4. **Ship** — **`audit`** until **`npm test`** green and surfaces list the tool consistently.

**Rule:** ingest **decisions and contracts**; **redact** secrets.

## Prioritization snapshot

Open work spanned **#76–#78**, memory **#68**, docs/evals **#70–#71**, docs site **#87**, and **#88–#91**. The group picked **[#89](https://github.com/danielsmithdevelopment/ClawQL/issues/89)** so the first enterprise slice **forced** shared plumbing: **`CLAWQL_ENABLE_*`**, Helm values, website tables — reusable for later **`metrics`** / **`governance`**.

## Why audit was the right vertical slice

**`audit`** is **optional**, **bounded** (ring buffer + max entries), and **inspectable** (`list` / `clear`) — it exercises the same **docs + chart + env** path as future enterprise tools without pretending to be **durable** vault storage. See **[enterprise-mcp-tools.md](https://github.com/danielsmithdevelopment/ClawQL/blob/main/docs/enterprise-mcp-tools.md)** for threat-model language.

## Optional tools: audit vs cache vs vault

| Tool                | Persists? | Use for                          |
| ------------------- | --------- | -------------------------------- |
| **`cache`**         | No        | Session scratch.                 |
| **`audit`**         | No        | Operator event trail in-process. |
| **`memory_ingest`** | Yes       | Durable narrative + recall.      |

## Helm and website wiring

Shipping **`audit`** required every **surface** that documents optional tools to stay in sync: **Helm** [`values.yaml`](https://github.com/danielsmithdevelopment/ClawQL/blob/main/charts/clawql-mcp/values.yaml), deployment template, **K8s** docs, and **docs.clawql.com** so operators see what clusters actually expose.
