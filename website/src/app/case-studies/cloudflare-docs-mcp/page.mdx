# Case study: `docs.clawql.com` with ClawQL MCP

This is an **end-to-end** account of shipping the documentation site to **Cloudflare Workers** at **`https://docs.clawql.com`** while using the same MCP tools you use for APIs: **`search`**, **`execute`**, **`memory_recall`**, and **`memory_ingest`**. It includes **control-plane** operations (accounts, zones, custom domains), **Worker runtime** failures (`fs`, MDX), and **edge caching** — not only “deploy succeeded.”

**Tracking:** [GitHub issue #87](https://github.com/danielsmithdevelopment/ClawQL/issues/87).

## Goals

1. Authenticate to **Cloudflare** and wire tokens for **Kubernetes** MCP, **Cursor** (HTTP MCP), and **Wrangler**.
2. **Deploy** the docs app (`website/`) to a **custom domain** with **OpenNext** + **Workers**.
3. **Operate** Cloudflare (Workers, domains) via **ClawQL** instead of ad-hoc **`curl`** where discovery and repeatability matter.
4. **Persist** setup, mistakes, and lessons in the **Obsidian vault** so later sessions do not repeat the same failures.

## Why Workers are a stress test for Next.js

The site is **Next.js 16** (App Router) on **Workers** via **OpenNext** — not a Node server. **Request-time** code must not assume **full Node `fs`**: **`unenv`** polyfills are incomplete; **`fast-glob`** and similar can call **`fs.readdir`**, which blew up at runtime with **1101** / **500** even when **`next build`** passed on a laptop. Separately, **edge `Cache-Control`** and **`public/_headers`** determine how quickly users see a fix after deploy.

## Mental model: account, zone, Worker, hostname

| Concept      | Role in this project                                                         |
| ------------ | ---------------------------------------------------------------------------- |
| **Account**  | **`account_id`** in REST paths; token needs **Account** read where required. |
| **Zone**     | DNS for **`clawql.com`** → **`zone_id`** for domain APIs.                    |
| **Worker**   | **`clawql-docs`** — OpenNext output **`.open-next/worker.js`**.              |
| **Hostname** | **`docs.clawql.com`** bound to that Worker via **Workers Domains** APIs.     |

When something breaks, decide: **Worker exception** (app code → **`wrangler tail`**) vs **API / DNS** (token scope, wrong **`zone_id`**, hostname conflict).

## Environment and stack

| Piece                             | Role                                                                                                                                                     |
| --------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ClawQL MCP**                    | Bundled **Cloudflare** provider for **`search`** / **`execute`**.                                                                                        |
| **`CLAWQL_CLOUDFLARE_API_TOKEN`** | Bearer for **`execute`**; must be on the **MCP process**, not only IDE `.env`.                                                                           |
| **`CLOUDFLARE_API_TOKEN`**        | Accepted alias — keep **one** token story across Wrangler + MCP.                                                                                         |
| **`memory_*`**                    | Require **`CLAWQL_OBSIDIAN_VAULT_PATH`** — see [memory-obsidian.md](https://github.com/danielsmithdevelopment/ClawQL/blob/main/docs/memory-obsidian.md). |
| **Website**                       | Next.js + **OpenNext**, Worker **`clawql-docs`**, **`NEXT_PUBLIC_SITE_URL=https://docs.clawql.com`** for canonical URLs.                                 |

## Wrangler vs REST execute

| Path                                   | Use when                                                                                                            |
| -------------------------------------- | ------------------------------------------------------------------------------------------------------------------- |
| **`npm run deploy`** in **`website/`** | You changed **MDX, layout, or Worker bundle** — primary artifact path.                                              |
| **`search` + `execute`**               | You need to **discover** `operationId`s, **attach** **`docs.clawql.com`**, or **audit** hostname → Worker bindings. |

**Wrangler success ≠ REST success** until token **scopes** are validated for the exact APIs you automate.

## Deploy script and custom domain

[`scripts/deploy-docs-to-cloudflare.sh`](https://github.com/danielsmithdevelopment/ClawQL/blob/main/scripts/deploy-docs-to-cloudflare.sh) runs **OpenNext build + deploy**, then **PUT**s the custom domain mapping. It uses **`CLAWQL_CLOUDFLARE_API_TOKEN`** or **`CLOUDFLARE_API_TOKEN`**, optional **`CLAWQL_DOCS_APEX_DOMAIN`**, **`CLAWQL_DOCS_HOSTNAME`**, **`CLAWQL_WORKER_NAME`**. Requires **`jq`**.

## How the four tools worked together

### `search()`

Discover **Workers / domains / routes** operations without loading the full Cloudflare OpenAPI into the model context.

### `execute()`

Call REST after **`operationId`** + path/body shape are known — e.g. **`workers.domains.update`**. **Path placeholders** (`account_id`, etc.) must match the spec; **body** fields (`hostname`, `service`, `zone_id`, …) must be complete.

### `memory_recall()`

At **session start** or when revisiting past deploy decisions: concrete **`query`**, reasonable **`limit`**, higher **`maxDepth`** when vault **wikilinks** tie topics together.

### `memory_ingest()`

After **decisions** (token strategy), **failures** (1101, `fs.readdir`), or **fixes** (static section maps): use **stable titles** and **`append: true`** for runbooks. **Never ingest secrets** — redact.

## End-to-end workflow

1. **Recall** vault notes on Cloudflare + MCP auth.
2. Set **`CLAWQL_CLOUDFLARE_API_TOKEN`** on the **MCP server** (and Wrangler env for deploy).
3. **`search`** / **`execute`** as needed for **domain** attachment and verification.
4. **Deploy** from **`website/`** with correct **`NEXT_PUBLIC_SITE_URL`**.
5. **Verify** the site + **`wrangler tail clawql-docs`**.
6. **Fix** Worker-safe code if the bundle throws.
7. **`memory_ingest`** the session.

## Failures and symptoms

| Symptom                                  | Likely cause                           | What helped                                        |
| ---------------------------------------- | -------------------------------------- | -------------------------------------------------- |
| Missing Cloudflare auth on **`execute`** | Token not on MCP process               | Set token on **server** env for stdio/HTTP MCP.    |
| **403**                                  | Token **IP allowlist**                 | Adjust egress policy.                              |
| API errors despite Wrangler              | **Narrow scopes**                      | Align **Account / Workers / DNS** with REST needs. |
| **500** / **1101**                       | Worker exception                       | **`wrangler tail`** — often **`fs`** on Workers.   |
| Prerender **`/concepts`** crash          | **`Children.only`** in **`CodePanel`** | MDX + Shiki multi-node **`code`**.                 |

## Fixes and verification

**Runtime:** Remove **request-time globbing** in **`layout.tsx`**; use static section metadata ([`home-page-sections.ts`](https://github.com/danielsmithdevelopment/ClawQL/blob/main/website/src/lib/home-page-sections.ts)). **MDX:** Normalize **`CodePanel`** children with **`Children.toArray`**.

**Verify:** HTTP **200** on **`https://docs.clawql.com/`**; **`wrangler tail`** clean on normal loads.

## Caching headers and purge

[`docs/website-caching.md`](https://github.com/danielsmithdevelopment/ClawQL/blob/main/docs/website-caching.md) documents **`next.config.mjs`** **`Cache-Control`** and **`public/_headers`**. Purge edge cache in Cloudflare if HTML looks stale after a deploy.

## Insights for future work

1. **Workers ≠ Node** — no **`fs.readdir`** on the hot path; **build-time** or **static maps**.
2. **Token parity** — validate **REST** scopes independently of **Wrangler**.
3. **Vault cadence** — **`memory_ingest`** after failures; **stable titles** + **`append: true`**.
4. **Case studies** — index in [`docs/case_studies/README.md`](https://github.com/danielsmithdevelopment/ClawQL/blob/main/docs/case_studies/README.md); **`memory_ingest`** canonical docs for later **`memory_recall`**.
