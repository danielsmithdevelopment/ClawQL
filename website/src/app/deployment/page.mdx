# Deployment

Production-style setups run **one** service: **`clawql-mcp-http`** exposes MCP at **`/mcp`** and GraphQL at **`/graphql`** on the same port. {{ className: 'lead' }}

## Streamable HTTP

From the published package:

```bash
PORT=8080 npx -p clawql-mcp clawql-mcp-http
```

Default local MCP endpoint: **`http://localhost:8080/mcp`**, health **`/healthz`**, Prometheus scrape **`GET /metrics`** (**`prom-client`** OpenMetrics). Your real URL depends on port, ingress, or tunneling—configure the client with **`url`**, not **`command`**.

**Native GraphQL/gRPC metrics:** scrape **`GET /metrics`** for OpenMetrics counters/gauges per **`sourceLabel`**. Optionally set **`CLAWQL_HEALTHZ_NATIVE_PROTOCOL_METRICS=1`** so **`GET /healthz`** also includes the same snapshot as JSON (**`nativeProtocolMetrics`**). Disable the metrics route only if required: **`CLAWQL_DISABLE_HTTP_METRICS=1`**. Repo: **[`docs/readme/deployment.md`](https://github.com/danielsmithdevelopment/ClawQL/blob/main/docs/readme/deployment.md)** ([#191](https://github.com/danielsmithdevelopment/ClawQL/issues/191)).

**Private tailnet:** if you are **new to Tailscale / Headscale**, start with **[Tailscale & Headscale](/tailscale)** (same content as **[`docs/deployment/tailscale-and-headscale-for-clawql.md`](https://github.com/danielsmithdevelopment/ClawQL/blob/main/docs/deployment/tailscale-and-headscale-for-clawql.md)**). For **managed Tailscale**, use HTTPS MagicDNS (for example **`https://your-machine.your-tailnet.ts.net/mcp`**) or **Serve**; for **self-hosted Headscale** and **`*.clawql.local`**, use runbook **[`docs/deployment/headscale-tailnet.md`](https://github.com/danielsmithdevelopment/ClawQL/blob/main/docs/deployment/headscale-tailnet.md)** ([#206](https://github.com/danielsmithdevelopment/ClawQL/issues/206)) and ACL starter **[`docs/deployment/headscale-acls-clawql.hujson`](https://github.com/danielsmithdevelopment/ClawQL/blob/main/docs/deployment/headscale-acls-clawql.hujson)** ([#213](https://github.com/danielsmithdevelopment/ClawQL/issues/213)). Keep **`CLAWQL_MCP_URL`** in env and reference **`${env:CLAWQL_MCP_URL}`** in **`mcp.json`**; align **`PAPERLESS_BASE_URL`**, **`ONYX_BASE_URL`**, **`CLAWQL_SANDBOX_BRIDGE_URL`**, and similar with the same network. Short form: **[`docs/readme/deployment.md`](https://github.com/danielsmithdevelopment/ClawQL/blob/main/docs/readme/deployment.md#private-tailnet-tailscale-magicdns)** ([#211](https://github.com/danielsmithdevelopment/ClawQL/issues/211)).

## Optional gRPC (protobuf MCP)

The same registered tools (including **`CallTool`**) can be exposed over **gRPC** in the **same process** as Streamable HTTP. Set **`ENABLE_GRPC=1`**; the server listens on **`GRPC_PORT`** (default **50051**) using the **[`mcp-grpc-transport`](https://www.npmjs.com/package/mcp-grpc-transport)** package (`model_context_protocol.Mcp`, `grpc.health.v1.Health`, optional reflection). Clients must send **`mcp-protocol-version`** metadata on protobuf RPCs. Invoking from the shell: **[`packages/mcp-grpc-transport` README — Testing and invoking MCP over gRPC](https://github.com/danielsmithdevelopment/ClawQL/blob/main/packages/mcp-grpc-transport/README.md#testing-and-invoking-mcp-over-grpc)**. On **Kubernetes**, the **`clawql-mcp-http`** Service publishes **HTTP** and **gRPC (50051)** so agents can reach **`localhost:50051`** (or the LoadBalancer IP) without port-forward when gRPC is enabled—see [Kubernetes](/kubernetes) and [`docs/deployment/deploy-k8s.md`](https://github.com/danielsmithdevelopment/ClawQL/blob/main/docs/deployment/deploy-k8s.md#service-ports-http-and-grpc).

## Security and supply chain

**Golden image (containers):** end-to-end narrative—repo gates → **single** BuildKit OCI export per image → **Trivy** on that layout → **`skopeo copy`** (same bytes) → **Cosign keyless** → **`imagetools`** promotion of **`latest`** / **`nightly`** — is in **[`docs/security/golden-image-pipeline.md`](https://github.com/danielsmithdevelopment/ClawQL/blob/main/docs/security/golden-image-pipeline.md)**. **Kubernetes:** the **Helm** chart defaults **Kyverno `verifyImages`** for the two GHCR ClawQL images ([Security](/security), **[`image-signature-enforcement.md`](https://github.com/danielsmithdevelopment/ClawQL/blob/main/docs/security/image-signature-enforcement.md)**). **npm** packages use a separate flow: **[`npm-supply-chain.md`](https://github.com/danielsmithdevelopment/ClawQL/blob/main/docs/security/npm-supply-chain.md)**. Operator **`cosign verify`** commands: [`docker/README.md`](https://github.com/danielsmithdevelopment/ClawQL/blob/main/docker/README.md).

## Cloud Run

One-shot script (sets project/region via env):

```bash
PROJECT_ID="your-project-id" \
REGION="us-central1" \
bash scripts/deploy/deploy-cloud-run.sh
```

Or with Make:

```bash
PROJECT_ID="your-project-id" REGION="us-central1" make deploy-cloud-run
```

After deploy, agents use **`https://<mcp-service-url>/mcp`**. Full options: [`docs/deployment/deploy-cloud-run.md`](https://github.com/danielsmithdevelopment/ClawQL/blob/main/docs/deployment/deploy-cloud-run.md).

For **remote** clusters, use Kustomize overlays with image tag injection:

```bash
ENV=dev IMAGE=us-central1-docker.pkg.dev/<project>/<repo>/clawql-mcp TAG=<tag> make deploy-k8s
```
